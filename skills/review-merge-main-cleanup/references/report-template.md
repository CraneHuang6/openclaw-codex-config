# 报告模板（Markdown）

## 1. 执行摘要

- 时间
- 仓库路径
- 基线分支
- 执行模式
- 清理范围
- 是否使用隔离 worktree

## 2. 已合并分支

- 分支列表（按执行顺序）

## 3. 已清理本地分支

- 分支列表
- 清理失败项与原因

## 4. 不确定分支完整审计

每个分支至少包含：

- 风险等级
- 风险原因
- 建议
- 提交列表
- 文件改动摘要（name-status）

## 5. 保留未合并分支

- 最终保留列表

## 6. 执行命令与关键结果

- 关键 git 命令
- 合并/清理结果（成功或失败）

## 7. 漂移检查

- 工作区状态签名（开始 vs 结束）
- 是否检测到非预期状态漂移
